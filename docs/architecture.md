# Architecture

## High-Level Model

The site is a static Jekyll blog with:

- Markdown content in `_posts` (single post-only publishing model).
- Liquid templates in `_layouts` and `_includes`.
- SCSS-driven design system in `_scss/_custom.scss`.
- Progressive-enhancement JavaScript in `assets/main.js`.

## Request-To-Render Flow

1. Jekyll loads `_config.yml`.
2. Collections and defaults resolve front matter.
3. Layouts and includes compose HTML output.
4. `assets/main.scss` compiles into `assets/main.css`.
5. Custom plugin `_plugins/image_optimizer.rb` losslessly/lossily compress assets into WebP.
6. Custom plugin `_plugins/git_metadata.rb` retrieves last-modified timestamps from Git history.
7. `assets/main.js` initializes UI behavior on `DOMContentLoaded`.

## View Layer Boundaries

- `_layouts/default.html`:
  - Outer shell, includes header/footer, loads assets.
- `_layouts/home.html`:
  - Chronological feed landing page.
- `_layouts/article.html`:
  - Article header, metadata, TOC container, share/comments panels.
- `_includes/header.html`:
  - Brand logo, compact search trigger/input, icon theme toggle.
- `_includes/footer.html` and `_includes/icons.html`:
  - Footer structure + social icons.

## Styling Architecture

- `assets/main.scss` imports:
  - Minima core.
  - `_scss/_custom.scss` overrides and complete custom layer.
- Design tokens originate in `_config.yml` under `design`.
- `_includes/head.html` publishes tokens as CSS variables.
- `_scss/_custom.scss` consumes semantic variables (`--bg`, `--accent`, `--heading`, etc).

## JavaScript Modules (Single File)

`assets/main.js` exposes function-level modules:

- Theme: `initThemeToggle`.
- Header search: `initHeaderSearch`.
- TOC: `initToc` (with ScrollSpy support).
- Code copy: `initCodeCopy`.
- Site search page: `initSearch`.
- Comments lazy load: `initDisqusLoader`.
- Reading progress: `initReadingProgress`.

All modules are defensive: each exits early if target DOM nodes do not exist.

## Data And Indexing

- `search.json` is built by Jekyll and consumed by `initSearch`.
- RSS and sitemap are generated by plugins (`jekyll-feed`, `jekyll-sitemap`).

## Deployment Architecture

- CI validates build and rendering.
- GitHub Pages workflow builds and deploys `_site` as artifact.
- No runtime server logic required post-deploy.
